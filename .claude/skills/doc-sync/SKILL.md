# 增量文档同步（/doc-sync）

本 skill 是 `/doc-sync-after-dev` 的增量模式快捷入口。

## 何时使用

- 开发过程中，发现实现偏离了需求或详设文档
- 对话中确认了需求变更（"改字段"、"加功能"、"这个不做了"）
- 对话中确认了实现逻辑变更（"接口改了"、"换方案"）
- 主动调用 `/doc-sync`

## 执行

读取并遵循 `doc-sync-after-dev/SKILL.md` 中的：

1. **文档格式统一规则**（规则 1-7）— 正文+changelog 分离、章节标记、废弃处理、版本号、旧格式迁移
2. **增量同步模式**（Step 1→2→3→4）— 识别变更 → 定位文档 → 更新文档 → 输出报告
3. **增量模式 checklist** — 执行完毕前逐项确认

## 与 /doc-sync-after-dev 的区别

| | /doc-sync | /doc-sync-after-dev |
|--|--|--|
| 时机 | 开发中随时 | 合并前一次 |
| 对比基准 | 对话上下文 | git diff |
| 范围 | 偏离的章节 | 全文档扫描 |
| 任务文档 | 不更新 | 更新 |
